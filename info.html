About This Tracker

This tool helps track Shiny, XXL, XXS, and Hundo (100% IV) collections in Pokémon GO.
One goal of this tracker is to help the user prepare for trades and complete their Pokédex (Dex).
Here are a few notes before moving forward:

  • All progress and changes are automatically saved locally in the browser as Pokémon are clicked on-screen.
  • Options to save data externally from the page are described below in the On the Page > Slide-Out Menu section.

A Very Short Background

The idea for this page is the result of a conversation with a Pokémon Community Ambassador (CA) in 2025. While it has had rolling requirements and many revisions, what is presented here is the culmination of lots of work.

On the Page

Main Tabs

There are four main tabs on the page: Shiny, XXL, XXS, Hundo. This page helps the user keep track of those Pokémon they have, and those they need to complete their Pokédex.

  • Shiny - A page of all the SHINY Pokémon, with all their forms and costumes.
  • XXL - Along with SHINY, XXL is one of the types recorded in, and required if the user is going to complete, the "Dex".
  • XXS - Like XXL, XXS is required to complete the Dex. Between the XXS and XXL, the collections can be very difficult to complete.
  • Hundo - The Holy Grail of Pokémon! Beside the SHINY Pokémon, the HUNDO is what everyone wants in this game!

Generation Tabs

There are currently nine generations of Pokémon currently available in Pokémon GO. The number of active Pokémon in the game is >1000, so navigating through the tabs, listed in the previous section, can be tedious. To help alleviate some of the scrolling, buttons for each generation have been provided under the main tabs. These generation buttons allow for a jump to the generation on the page; they do not stay highlighted, as they are simply jump points.

Individual Pokémon Cells

As mentioned above, the intent of this page is to help the user complete their Dex, and promote trading by sharing visual lists of "needs". To that end, the cells on this page provide a clickable means to visualize the user's collection.

  • Each Pokémon gets its own cell on each tab on the page.
  • Each cell (Pokémon) has four states: Grey (default), Yellow, Blue, and Red.
  • These "states" can mean whatever the user wants them to mean. In the How to Use section, there are examples of potential use scenarios.
  • The clicked state of each cell is separate for each of the four tabs discussed above.
  • The states are saved in the browser for later recall, edits, and sharing, (see next section).

Slide-Out Menu

There is a slide-out menu on the left of the page, activated with the hamburger icon (☰) in the upper left corner of the page. On that menu are a number of buttons, explained here.

• Share
This button allows the user to share the contents (selected cells) of one of the filters on one of the tabs, and the use will be prompted for the filter color to share. This button will create a "short URL" that will automatically be copied to the user's clipboard and may be shared with others (think potential trading, bragging about a collection, etc.). Note that this "sharing" option is read only for anyone accessing the URL. This is the only non-destructive save or sharing option available, and it's safe for sharing with others; they will not have access to your actual collection. There it's currently a 30 day lifespan on these URLs, which may change depending on overall usage.
• Save
To make the save more permanent, the "Save" button will create a very long URL that, when accessed, will load the contents of the URL, replacing whatever is currently saved in the browser. This is helpful for point-in-time saves or quick transfers between browsers. This is a destructive URL and will replace current contents.
• Clear
Simple, but destructive, clearing of all data currently in the browser. Allows for a clean start on all tabs in cases where this is needed. Imagine having to "unclick" potentially hundreds of cells to start over! This is the safety net for that type of situation.
• Export and Import
These buttons are another more permanent way to allow the user to save their configuration and import it elsewhere. This method is the best and safest way to create a save of the data. The JSON file will be downloaded to the user's device to be available for "import" to other browsers, or sent to other devices for use elsewhere.
• Info
The Info button how this page/information was displayed. Simple yet hopefully effective guidance for the tool.

How to Use

There is no "right" or "wrong" way to think about or use this tool. The user is completely free to define their own manner of use. Below are some examples for how the tool could be used to visualize a collection.

Example 1: "Caught" This example uses the filters as "caught indicators", meaning that the Pokémon clicked have been caught (can be seen in the in-game inventory and/or Dex).
  1. Choose a tab; for this example SHINY is selected.
  2. Click a Pokémon to cycle its status color.
    • Grey: Uncaught
    • Yellow: Caught
    • Blue: Traded (also caught)
    • Red: Lucky / Special (also caught)
  3. Share the grey filter with others as "needs" for your Shiny Dex.
Example 2: "Need" This example is different from the the first, in that the user clicks on Pokémon they want or need, rather than those they have caught. In this example, the actual filter color used is of no importance, but only one would be used, at least in theory, if the user is creating a "needs" list with the filter.
  1. Choose a tab; this example is using the Hundo tab and the YELLOW filter.
  2. Click a Pokémon once to highlight with yellow status color.
  3. Share the yellow filter with others as "needs" for your Hundo Dex.
